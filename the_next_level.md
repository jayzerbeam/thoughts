# The Next Level

In the beginning, programming was physical. Changing functionality of a computer meant physically flipping switches, patching cables, punching cards, etc.

But Surely things could be better... And so assembly was born.
Now programming could be done with instructions while the circuits, switching, and cables were abstracted away.
Was understanding how the hardware worked necessary to program? No.
Did knowing make you a better programmer? Yes.

But surely things could be better... And so high level languages were born.
Now programming could focus more clearly on logic while the registers, memory, and instructions were abstracted away.
Was understanding machine architecture necessary to program? No.
Did knowing make you a better programmer? Yes.

But surely things could be better... And so libraries and packages were born.
Now programming could use other programmer's code, various logic was abstracted away.
Was understanding an underlying implementation necessary to program? No.
Did knowing make you a better programmer? Yes.

But surely things could be better... And so code generating LLMs code were born.
Now programming didn't involve programming at all, programming itself is abstracted away!
Was understanding how to program necessary to program? Hrmmm...

-----

There's something to be said about having depth of knowledge. Greater depth yields better results. This has been true, always.

But that last level... Not only has the generational gap become so vast, it is fundamentally different.

All levels previous to LLMs are predictable and exact. An assembly instruction produces an exact calculation. Compiling a program produces specific bytecode. Prompting an LLM... Well, we may get what we want, but how that is achieved is opaque.

All levels previous to LLMS are scrutable and tractable. A faulty wire can be found and repaired. A program error can be debugged and fixed. An LLM hallucination... Well, hopefully the output goes through a human, and they possess the depth of knowledge to catch it.
